# Practical 10.3 — Parallel Research and Aggregation

## Build

Split a research question into independent subtopics, run researchers in parallel, and aggregate structured results.

## Practice

Add per-task timeouts, partial-result handling, source deduplication, and conflict detection. Do not let one failed branch discard successful evidence.

## Success checklist

- [ ] Parallel tasks have independent status.
- [ ] Duplicate sources are merged.
- [ ] Conflicting claims are flagged for review.
