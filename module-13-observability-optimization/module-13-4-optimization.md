# Practical 13.4 — Optimize Cost and Latency

## Practice

Establish a baseline, then try model routing, prompt reduction, caching, parallelism, or fewer reflection steps. Re-run the evaluation suite after every change.

## Success checklist

- [ ] Before/after cost and latency are reported.
- [ ] Quality remains within the agreed threshold.
- [ ] Cache keys include relevant model and prompt versions.
- [ ] Sensitive responses are not cached unsafely.
