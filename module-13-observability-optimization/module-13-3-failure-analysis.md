# Practical 13.3 — Diagnose Failures Safely

## Practice

Inject a model timeout, malformed structured output, unavailable MCP server, empty retrieval, and failed workflow step. Diagnose each from traces and structured logs.

Create a short runbook with symptom, query, likely cause, mitigation, and escalation.

## Success checklist

- [ ] Clients receive safe errors.
- [ ] Operators retain useful context.
- [ ] Every injected failure has a runbook entry.
