# Practical 11.1 — Build a Golden Evaluation Dataset

## Build

Create at least 30 versioned cases covering normal success, ambiguity, refusal, tool failure, unanswerable questions, and adversarial input.

Define expected facts, allowed tools, forbidden actions, and maximum budgets without requiring exact wording.

## Success checklist

- [ ] Cases represent real user tasks.
- [ ] Critical safety cases are labeled.
- [ ] Dataset changes receive version numbers.
