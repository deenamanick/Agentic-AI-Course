# Practical 11.2 — Test Tools and Execution Trajectories

## Why

A correct final sentence can hide an unsafe or wasteful execution path.

## Build

Assert selected tools, argument schemas, call order, handoffs, retries, and stop conditions. Mock external services for deterministic tests.

## Success checklist

- [ ] Unauthorized tools cause failure.
- [ ] Arguments are validated.
- [ ] Loop and retry limits are tested.
- [ ] Side effects are idempotent.
