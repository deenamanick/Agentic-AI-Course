# Practical 15.2 — Design Architecture and Security

## Build

Create a container diagram, data flow, tool contracts, job state machine, threat model, authorization model, memory policy, and human-approval points.

## Success checklist

- [ ] The architecture follows Durable Orchestrator.
- [ ] Trust boundaries and sensitive data are marked.
- [ ] Consequential writes require approval.
- [ ] Recovery and deletion paths are designed.
