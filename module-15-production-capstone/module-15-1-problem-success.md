# Practical 15.1 — Select the Problem and Define Success

## Build

Choose a real user, painful task, and bounded outcome. Create user stories, non-goals, a single-agent baseline, evaluation cases, quality thresholds, and latency/cost budgets.

## Success checklist

- [ ] The problem needs tools, retrieval, memory, or durable work.
- [ ] Success is measurable.
- [ ] A simpler non-agent solution was considered.
