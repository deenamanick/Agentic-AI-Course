# Practical 15.4 — Release, Demonstrate, and Defend

## Demonstration

Show the happy path, an unanswerable request, prompt injection, denied access, human approval, tool failure, cancellation, and recovery after interruption.

Submit architecture, threat model, API/tool docs, evaluation report, runbook, and cost report.

## Success checklist

- [ ] A request can be diagnosed by ID.
- [ ] Recovery does not duplicate effects.
- [ ] Known limitations are documented.
- [ ] Reviewers can reproduce the evaluation.
