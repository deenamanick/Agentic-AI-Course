# Practical 4.3 — Plan, Execute, and Finalize

## Build

Inspect how the general path creates a plan, executes it, and finalizes an answer. Limit plan length and make failed steps visible.

## Practice

Compare a simple direct request with a multi-step planning request. Identify when planning adds unnecessary overhead.

## Success checklist

- [ ] Plans use a validated structure.
- [ ] Plan and execution lengths are bounded.
- [ ] Finalization reports incomplete work honestly.
