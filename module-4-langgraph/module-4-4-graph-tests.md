# Practical 4.4 — Trace and Test the Graph

## Practice

Run `scripts/test_graph.sh`, then add cases for every route, node failure, invalid state, model timeout, and stop condition. Trace node names and durations.

## Success checklist

- [ ] Every edge and terminal path has a test.
- [ ] A failed node produces a controlled result.
- [ ] Traces reveal the selected path.
- [ ] No test depends on hidden global state.
