# Practical 4.2 — Add Conditional Routing

## Build

Route requests to math or general paths using a structured decision. Add an `unknown` or fallback path.

## Practice

Test obvious math, ordinary conversation, ambiguous text, and malformed router output.

## Success checklist

- [ ] Routes are drawn as a simple diagram.
- [ ] Router output is validated.
- [ ] Every possible route reaches a terminal state.
